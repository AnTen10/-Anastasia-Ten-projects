## Структура Data Lake

Используется трёхслойная архитектура:

### 1. Raw слой (сырые данные)

- Путь: `/user/dudosaa/data/geo/events`
- Источник: `/user/master/data/geo/events`
- Формат: Parquet
- Обновление: ежедневно

Описание:
- Данные копируются из исходного источника без изменений
- Слой является неизменяемым
- Используется как источник истины

---

### 2. Processed слой (подготовленные данные)

- Путь: `/user/dudosaa/data/processed/`
- Формат: Parquet

Описание:
- Распаковка вложенной структуры `event.*`
- Отбор необходимых полей
- Очистка данных (фильтрация null-значений)
- Добавление и преобразование полей:
  - координаты
  - временные метки
  - timezone (на основе справочника городов)
- Джойн с гео-справочником (`geo_2.csv`)
- Возможна агрегация и нормализация данных

---

### 3. Analytics слой (витрины данных)

- Путь: `/user/dudosaa/data/analytics/`
- Формат: Parquet

Описание:
- Готовые датасеты для аналитиков и продуктовых задач
- Используются для:
  - рекомендаций друзей
  - геоаналитики пользователей
  - анализа активности (сообщения, лайки, подписки)
  - анализа регистраций пользователей
  - анализа перемещений пользователей

---

# Примечание:
Домашний адрес определяется как город, в котором пользователь находился не менее 27 дней подряд.
В текущем датасете такие пользователи отсутствуют (максимум — 3 дня), поэтому поле home_city принимает значение null.
