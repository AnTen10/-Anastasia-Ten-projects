# Data Lake for Social Network Geo Analytics

## О проекте

В рамках проекта был доработан Data Lake социальной сети с использованием Apache Spark и Apache Airflow. Основной целью стало построение новых аналитических витрин на основе геоданных пользователей и автоматизация их ежедневного обновления.

Проект включает обработку событий пользователей, определение географического положения, построение пользовательских и географических витрин, а также формирование рекомендаций друзей.

---

## Бизнес-задача

Необходимо было решить несколько задач:

- расширить существующий Data Lake геоданными;
- определить города пользователей по координатам сообщений;
- построить аналитические витрины для пользователей и геозон;
- реализовать систему рекомендаций друзей;
- автоматизировать ежедневное обновление витрин с помощью Airflow.

## Что реализовано

### Организация Data Lake

Разработана трёхслойная архитектура хранения данных:

- Raw Layer;
- Processed Layer;
- Analytics Layer.

### Пользовательская витрина

Построена витрина пользователей, содержащая:

- текущий город пользователя;
- домашний город;
- количество посещённых городов;
- историю перемещений;
- локальное время последнего события.

### Географическая витрина

Рассчитаны агрегированные показатели по городам:

- сообщения;
- реакции;
- подписки;
- регистрации пользователей;
- недельная и месячная статистика.

### Витрина рекомендаций друзей

Реализована логика поиска потенциальных друзей:

- общий канал;
- отсутствие переписки между пользователями;
- нахождение пользователей в одной географической зоне.

### Автоматизация

Разработан DAG Apache Airflow для последовательного запуска всех этапов построения витрин.
