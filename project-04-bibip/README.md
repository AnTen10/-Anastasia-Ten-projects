# BiBiP File-Based Car Storage

## О проекте

Проект представляет собой файловое хранилище данных для автосалона «Бибип», реализованное на Python.

Вместо использования СУБД данные сохраняются в текстовых файлах.

Для поиска записей используются индексные файлы вида:

*ключ → номер строки*

Индексы загружаются в память целиком, после чего доступ к данным осуществляется через номер строки без полного сканирования файла.

Основная задача проекта — реализовать сервис хранения автомобилей, моделей и продаж с поддержкой операций обновления, поиска и аналитики.

---

## Возможности проекта

Реализован сервис `CarService`, который позволяет:

- хранить модели автомобилей;
- хранить автомобили;
- оформлять продажи автомобилей;
- получать список автомобилей по статусу;
- получать полную информацию об автомобиле;
- изменять VIN автомобиля;
- отменять продажу автомобиля;
- получать ТОП-3 самых продаваемых моделей.

## Модель данных

### Cars

- vin — идентификатор автомобиля
- model — ссылка на модель
- price — закупочная стоимость
- date_start — дата поступления
- status — статус автомобиля

### Models

- id — идентификатор модели
- name — название модели
- brand — бренд

### Sales

- sales_number — номер продажи
- car_vin — VIN автомобиля
- cost — стоимость продажи
- sales_date — дата продажи

## Реализованные задачи

### Работа с данными

- добавление моделей автомобилей;
- добавление автомобилей;
- оформление продажи;
- изменение статуса автомобиля после продажи;
- обновление VIN;
- отмена продажи.

### Поиск

- получение автомобилей по статусу;
- получение полной информации об автомобиле путём объединения данных из файлов моделей, автомобилей и продаж (аналог SQL JOIN).

### Аналитика

- определение трёх самых продаваемых моделей автомобилей.

---

## Особенности реализации

Данные организованы в отдельных текстовых файлах, для которых создаются индексные файлы, содержащие соответствие между первичным ключом и номером строки. Такой подход позволяет быстро находить записи без полного сканирования файлов и имитирует работу простейшей файловой СУБД.

## Итоги

Разработан файловый сервис хранения данных для автосалона, реализующий:

- хранение данных без использования СУБД;
- работу с индексными файлами;

- поиск по ключевым полям;
- аналитику продаж моделей автомобилей.

Проект демонстрирует принципы построения файловой системы хранения данных, организации индексного доступа и реализации базовых операций СУБД средствами Python без использования внешней базы данных.
