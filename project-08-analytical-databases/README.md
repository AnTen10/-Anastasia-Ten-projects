# ПAnalytical Databases

## О проекте

Проект посвящён расширению аналитического хранилища данных на основе подхода Data Vault. В рамках работы был реализован полный цикл загрузки нового источника данных, построения моделей хранения и подготовки аналитической витрины для оценки активности пользователей в сообществах социальной сети.

---

## Бизнес-задача

Основная бизнес-задача — определить сообщества с наибольшей конверсией участников в первое сообщение, чтобы маркетинговая команда могла выбирать наиболее эффективные площадки для размещения рекламы.

## Что реализовано

### ETL-процесс
- загрузка файла group_log.csv из S3;
- автоматизация процесса с использованием Apache Airflow;
- подготовка данных с помощью Pandas;
- загрузка данных в слой STAGING аналитического хранилища Vertica.

### Data Vault

Созданы объекты хранилища данных:

- Link l_user_group_activity;
- Satellite s_auth_history.

Реализовано наполнение таблиц с использованием данных из STAGING и существующих Hub-таблиц.

### Аналитика

Подготовлены SQL-запросы для:

- расчёта количества участников групп;
- определения пользователей, написавших сообщения;
- вычисления конверсии пользователей в первое сообщение;
- формирования рейтинга групп по уровню вовлечённости.

# Итоги

В результате проекта реализован полный цикл обработки нового источника данных: автоматическая загрузка файлов из S3, наполнение аналитического хранилища по модели Data Vault и построение SQL-запросов для оценки вовлечённости пользователей в сообщества социальной сети. Полученные данные позволяют маркетинговой команде выбирать группы с наиболее высокой конверсией участников в активное общение.
