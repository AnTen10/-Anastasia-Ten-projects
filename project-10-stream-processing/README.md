# Stream Processing for Restaurant Promotions

## О проекте

Проект посвящён разработке сервиса потоковой обработки данных для автоматической подготовки push-уведомлений о рекламных акциях ресторанов.

Приложение получает события из Apache Kafka, фильтрует только активные рекламные кампании, сопоставляет их с подписчиками ресторанов из PostgreSQL и формирует два независимых потока данных:

сообщения для сервиса push-уведомлений (Kafka);
записи для последующего анализа пользовательского отклика (PostgreSQL).

Проект реализован с использованием **PySpark Structured Streaming, Apache Kafka** и **PostgreSQL**.

---

## Бизнес-задача

Необходимо автоматизировать процесс информирования пользователей о временных рекламных акциях ресторанов.

Для каждой поступившей акции сервис должен:

- получить сообщение из Kafka;
- определить, действует ли акция в текущий момент;
- найти всех пользователей, подписанных на ресторан;
- сформировать персонализированные события для отправки push-уведомлений;
- сохранить информацию о созданных уведомлениях в базе данных для последующей аналитики.

## Что реализовано

### Потоковая обработка

- чтение сообщений из Kafka;
- десериализация JSON-сообщений;
- определение схемы данных;
- фильтрация активных рекламных кампаний по времени.

### Интеграция с PostgreSQL

- загрузка справочника подписчиков;
- объединение потоковых и статических данных;
- запись результатов в таблицу `subscribers_feedback`.

### Отправка уведомлений

Для каждого пользователя, подписанного на ресторан:

- формируется событие;
- добавляется время создания триггера;
- сообщение сериализуется в JSON;
- отправляется в выходной Kafka-топик.

### Обработка микробатчей

Используется механизм `foreachBatch`, внутри которого:

- выполняется кэширование датафрейма;
- запись данных в PostgreSQL;
- публикация сообщений в Kafka;
- очистка памяти после обработки.
