# Data Warehouse

## Описание проекта

В рамках проекта выполнено расширение корпоративного хранилища данных (DWH) путем подключения нового источника данных `external_source` и реализации инкрементальной витрины заказчиков.

Проект демонстрирует построение ETL-процесса с использованием SQL, включающего загрузку измерений, фактов и последующее формирование аналитических витрин.

---

## Цель проекта

Интегрировать новый источник данных `external_source` в корпоративное хранилище данных (DWH) и реализовать инкрементальную витрину `customer_report_datamart`, содержащую агрегированную статистику по заказчикам за каждый отчетный месяц.

Витрина предназначена для:

- программы лояльности;
- анализа поведения клиентов;
- персональных рекомендаций;
- маркетинговой аналитики.

## Используемые источники данных

Проект объединяет данные из четырех источников:

- `source1`
- `source2`
- `source3`
- `external_source`

Все данные предварительно объединяются во временную таблицу `tmp_sources`, после чего загружаются в корпоративное хранилище.

Источник `external_source` содержит две таблицы:

- `craft_products_orders` — сведения о мастерах, товарах и заказах;
- `customers` — информация о заказчиках.

Данные нового источника объединяются с существующими посредством `UNION ALL` и загружаются в DWH.

## Реализованный ETL-процесс

Для загрузки измерений и фактов используются операции `MERGE`, позволяющие выполнять вставку новых записей и обновление существующих.

---

### Создание служебных таблиц

Скрипты:

- `DDL_create_tables.sql`
- `DDL_customer_report_datamart.sql`

Создает:

- витрину `craftsman_report_datamart`;
- таблицу `load_dates_craftsman_report_datamart`, используемую для хранения даты последней успешной инкрементальной загрузки витрины мастеров.
- витрину `customer_report_datamart`;
- таблицу `load_dates_customer_report_datamart`.

Служебные таблицы `load_dates_*` используются для хранения даты последней успешной инкрементальной загрузки соответствующей витрины.

---

### Загрузка измерений

Скрипт:

`DML_load_dimensional_tables.sql`

Выполняет:

- объединение данных из всех источников (`UNION ALL`);
- загрузку измерений:
  - `d_craftsman`;
  - `d_customer`;
  - `d_product`;
- синхронизацию данных с помощью `MERGE`.

---

### Загрузка таблицы фактов

Скрипт:

`DML_load_fact_table.sql`

Выполняет:

- формирование временной таблицы `tmp_sources_fact`, содержащей ссылки на записи измерений (`d_craftsman`, `d_customer`, `d_product`);
- загрузку таблицы фактов `f_order`;
- обновление существующих записей таблицы фактов при изменении информации о заказе (`order_completion_date`, `order_status`, `load_dttm`).

---

### Витрина мастеров

Скрипт:

`DML_craftsman_report_datamart.sql`

Рассчитывает инкрементальную витрину по мастерам.

Основные показатели:

- доход мастера;
- доход платформы;
- количество заказов;
- средняя стоимость заказа;
- средний возраст клиентов;
- медианное время выполнения заказа;
- наиболее популярная категория товаров;
- распределение заказов по статусам.

---

### Витрина заказчиков

DDL:

`DDL_customer_report_datamart.sql`

Создает:

- витрину `customer_report_datamart`;
- таблицу `load_dates_customer_report_datamart`, используемую для хранения даты последней успешной инкрементальной загрузки витрины заказчиков.

DML:

`DML_customer_report_datamart.sql`

Реализует инкрементальное обновление витрины заказчиков.

Для каждого клиента рассчитываются:

- общая сумма заказов;
- доход платформы;
- количество заказов;
- средняя стоимость заказа;
- средний возраст мастеров;
- медианное время выполнения заказа;
- наиболее популярная категория товаров;
- наиболее популярный мастер;
- количество заказов по каждому статусу;
- отчетный период.

## Использованные возможности SQL

В проекте применены:

- `MERGE`;
- `CTE (WITH)`;
- `UNION ALL`;
- оконные функции (`ROW_NUMBER`, `RANK`);
- агрегатные функции (`SUM`, `AVG`, `COUNT`);
- `CASE WHEN`;
- `PERCENTILE_CONT`;
- инкрементальная загрузка по `load_dttm`;
- временные таблицы (`TEMP TABLE`).

---

## Инкрементальная загрузка

Все расчеты витрин выполняются инкрементально.

Для определения изменившихся данных используется поле `load_dttm` таблиц измерений и фактов. В обработку попадают только записи, изменившиеся после последней успешной загрузки.

После завершения обновления дата загрузки фиксируется в специальных таблицах:

- `load_dates_craftsman_report_datamart`;
- `load_dates_customer_report_datamart`.

Это позволяет избежать полного пересчета витрин и существенно сократить время выполнения ETL-процесса.

## Особенности реализации

Проект реализован с использованием инкрементальной обработки данных.

При расчете витрин используются:

- CTE (`WITH`);
- операции `MERGE`;
- оконные функции (`ROW_NUMBER`, `RANK`);
- агрегатные функции (`SUM`, `AVG`, `COUNT`);
- `CASE WHEN` для расчета количества заказов по статусам;
- `PERCENTILE_CONT` для вычисления медианного времени выполнения заказа;
- временные таблицы (`TEMP TABLE`) для промежуточной обработки данных.
