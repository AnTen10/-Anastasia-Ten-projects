# Gastro Hub Data Modeling and SQL Analytics

## О проекте

В рамках проекта разработана реляционная база данных для сети ресторанов Gastro Hub. Выполнено проектирование нормализованной схемы данных, загрузка информации из исходной схемы, реализация аналитических представлений и транзакционных операций средствами PostgreSQL. Проект демонстрирует применение возможностей PostgreSQL для моделирования данных, аналитики и безопасного изменения данных.

## Что реализовано

- спроектирована нормализованная схема базы данных;
- реализованы связи между сущностями посредством PK/FK;
- выполнена загрузка данных из raw_data;
- созданы представления и материализованные представления;
- разработаны аналитические SQL-запросы;
- реализованы транзакционные операции с использованием механизмов блокировок PostgreSQL;
- выполнена обработка полуструктурированных данных JSONB.

## Этапы реализации

### Проектирование базы данных
- создание схемы cafe;
- определение структуры таблиц;
- настройка первичных и внешних ключей;
- использование пользовательского типа ENUM.

### Загрузка данных
- перенос информации из схемы raw_data;
- преобразование исходных данных;
- сохранение связей между сущностями.

### Аналитика

Реализованы SQL-запросы для решения следующих задач:
- определение ТОП-3 заведений по среднему чеку;
- анализ динамики среднего чека по годам;
- анализ смен менеджеров;
- работа с меню в формате JSONB;
- поиск самых дорогих позиций меню;
- массовое обновление цен;
- изменение структуры таблицы в транзакции.
