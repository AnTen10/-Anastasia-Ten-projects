# Car Dealership Database Design and SQL Analytics

## О проекте

Проект посвящён проектированию реляционной базы данных в PostgreSQL. Исходный CSV-файл с историей продаж автомобилей был преобразован в нормализованную структуру (3NF), после чего подготовлены SQL-запросы для анализа продаж.

---

## Бизнес-задача

Необходимо было решить две основные задачи:

- преобразовать ненормализованные данные в структуру, соответствующую требованиям третьей нормальной формы (3NF);
- подготовить аналитические запросы для оценки продаж автомобилей.

## Модель данных

В результате проектирования были выделены следующие сущности:

| Таблица | Назначение |
|----------|------------|
| countries | страны происхождения брендов |
| brands | производители автомобилей |
| models | модели автомобилей |
| colors | справочник цветов |
| customers | покупатели |
| sales | информация о продажах |

Связи между таблицами реализованы с использованием первичных и внешних ключей.

## Что реализовано

### Загрузка данных

- создание схем `raw_data` и `car_shop`;
- импорт исходных данных из CSV;
- заполнение нормализованных таблиц.

### Проектирование базы данных

Спроектирована реляционная модель данных с использованием:

- первичных ключей;
- внешних ключей;
- ограничений `NOT NULL`;
- ограничений `UNIQUE`;
- нормализации до третьей нормальной формы (3NF).

### Обработка данных

При переносе данных реализованы:

- разбор составных полей;
- устранение дублирования информации;
- распределение данных между сущностями;
- построение связей между таблицами.

### Аналитические запросы

Подготовлены SQL-запросы для получения бизнес-метрик:

- процент моделей без значения расхода топлива;
- средняя стоимость автомобилей по брендам и годам;
- средняя стоимость автомобилей по месяцам;
- список приобретённых автомобилей по каждому покупателю;
- максимальная и минимальная стоимость автомобилей по странам происхождения брендов;
- количество покупателей из США.

## Особенности реализации

При проектировании модели особое внимание уделено устранению избыточности данных.

Исходная таблица содержала составные атрибуты (марка, модель, цвет автомобиля), которые были выделены в отдельные сущности. Это позволило привести модель к третьей нормальной форме (3NF) и исключить дублирование информации.

Наполнение схемы реализовано средствами SQL без использования внешних ETL-инструментов.
