# Airflow ETL Pipeline

## О проекте

Проект посвящён разработке ETL-пайплайна на базе Apache Airflow для автоматической загрузки данных, построения витрин данных и расчёта аналитических показателей.

В рамках проекта был доработан существующий ETL-процесс:

- реализована поддержка новых статусов заказов (`shipped`, `refunded`);
- обеспечена обратная совместимость со старыми инкрементами;
- построена витрина `mart.f_sales`;
- разработана витрина `mart.f_customer_retention`;
- автоматизирован весь процесс загрузки данных через Airflow DAG.

---

## Что реализовано

Проект включает:

- получение данных через REST API;
- ожидание готовности отчёта;
- загрузку инкрементов из S3;
- загрузку данных в слой staging PostgreSQL;
- обновление справочников;
- построение витрины продаж;
- расчёт витрины Customer Retention;
- автоматический запуск всех этапов через Airflow.

## ETL-процесс

Пайплайн выполняет следующие этапы:

1. генерация отчёта через API;
2. ожидание окончания формирования отчёта;
3. получение очередного инкремента;
4. загрузка CSV-файлов из S3;
5. загрузка данных в слой **staging PostgreSQL**;
6. обновление справочников (`d_item`, `d_customer`, `d_city`);
7. загрузка фактов продаж (`f_sales`);
8. создание (при необходимости) витрины Customer Retention;
9. расчёт метрик удержания клиентов.

---

## Реализованные бизнес-требования

### Поддержка статусов заказов

В витрине `mart.f_sales` реализована обработка двух статусов:

- `shipped`
- `refunded`

Для возвратов сумма продажи записывается со знаком минус, что позволяет корректно рассчитывать итоговую выручку.

Также реализована обратная совместимость со старым форматом данных — при отсутствии поля `status` автоматически устанавливается значение `shipped`.

---

## Витрина Customer Retention

Автоматически рассчитываются показатели:

- количество новых клиентов;
- количество вернувшихся клиентов;
- количество клиентов, оформивших возврат;
- доход от новых клиентов;
- доход от постоянных клиентов;
- сумма возвращённых средств.

Расчёт выполняется в недельном разрезе (`weekly`) по категориям товаров.

## Итоги

В результате проекта был разработан автоматизированный ETL-процесс, который получает данные из внешнего API, загружает их в PostgreSQL, формирует витрину продаж и рассчитывает показатели удержания клиентов.
